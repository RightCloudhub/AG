想做这个项目，是因为企业里真正难答的问题往往是跨实体、跨文档的多跳关系推理，而传统向量 RAG 只能召回相似片段却拼不出推理链，纯 GraphRAG 虽有结构却策略静态、不会追问；因此你要把知识图谱当作可导航的结构化记忆、把 Agent（规划—执行—反思—护栏）当作决策大脑，在成本可控、可审计、证据不足就诚实兜底的前提下，做出一套显著优于基线、面向投研/风控等知识密集型场景的 Agentic GraphRAG


一句话概括
传统向量 RAG 答不好跨实体、跨文档的多跳问题；本项目用「知识图谱做结构化记忆 + Agent 做动态规划/检索/反思」，在可审计、成本可控的前提下，显著提升多跳问答的准确率与可解释性。

面试可用金句
「我不是在做更好的检索，而是在做会沿图谱追问的推理系统——让 Agent 在约束下把分散证据走成可审计的答案链。」
