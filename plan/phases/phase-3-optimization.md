# 阶段三：工程优化（3-4周）

**目标**：性能、成本、质量治理达到上线水位 —— 效果最终达标（AC-1/2）、延迟达标（AC-4）、护栏与预算生效（AC-6）、增量更新可用（AC-5）。
**前提**：G2 通过。

## 周计划

| 周 | 重点 |
|---|---|
| W1 | 复杂度分诊 + 检索并行化 + 融合排序 |
| W2 | 缓存 + 流式响应 + 监控/预算控制 |
| W3 | 增量更新 + 实体消歧 + 质量治理闭环 |
| W4 | 全量回归评测 + 压测 + 护栏专项测试 + G3 评审 |

## 任务清单

### 性能与成本（PERF）
- [ ] `P3-PERF-01` 复杂度分诊器：规则 + 轻量 LLM 判定，简单问题走 Fast Path（FR-AG-01）；分诊准确性在评测集上验证（误分诊导致的 Accuracy 损失 <2pp）
- [ ] `P3-PERF-02` 三路检索并行执行（FR-RT-05）
- [ ] `P3-PERF-03` RRF 融合排序，预留 Re-ranker 接口（FR-RT-04）
- [ ] `P3-PERF-04` 中间结果缓存：embedding 缓存、子问题级检索结果缓存、热点问题答案缓存（含失效策略：图谱更新后相关缓存失效）
- [ ] `P3-PERF-05` LLM 分级用模：Planner/Critic 用强模型，Executor 汇总/分诊用轻模型（开放问题 #5 落地）
- [ ] `P3-PERF-06` SSE 流式响应：推送当前子问题、当前跳数、最终答案（FR-API-02）
- [ ] `P3-PERF-07` 压测：并发场景下 Fast Path P95 ≤3s、Agentic P95 ≤8s（NFR-01/AC-4）

### 成本护栏与运营（OP）
- [ ] `P3-OP-01` 查询级指标采集：延迟分布、跳数分布、LLM 调用次数、token 成本、工具调用明细（FR-OP-01）
- [ ] `P3-OP-02` 预算控制：租户/用户/单查询三级成本上限，超限熔断返回明确错误码（FR-OP-02）
- [ ] `P3-OP-03` 全链路 trace：查询→子问题→工具→LLM 调用，查询 ID 串联（NFR-08）
- [ ] `P3-OP-04` 护栏专项测试集：发散问题、循环诱导问题、超长问题，验证触顶兜底与成本上限（AC-6）

### 图谱治理（KG）
- [ ] `P3-KG-01` 增量更新流程：新文档→抽取→冲突检测→高置信自动合并/低置信入审核队列→索引同步（FR-KG-05），更新期间线上查询不中断（AC-5 演练）
- [ ] `P3-KG-02` 实体消歧：别名归一 + 嵌入相似度候选 + LLM 判定合并；低置信入人工审核（FR-KG-04）
- [ ] `P3-KG-03` 质量治理界面（最小版）：抽样审核、对错标注、按置信度过滤（FR-KG-06）
- [ ] `P3-KG-04` 知识管理 API：文档上传、任务状态、审核队列操作（FR-API-03）

### 可解释性（AN）
- [ ] `P3-AN-01` 推理链落库 + 按查询 ID 审计回查 API（FR-AN-04）
- [ ] `P3-AN-02` 推理链可视化组件：分解树 + 图路径 + 引用列表，节点可点击查看原文（FR-AN-03，为阶段四界面复用）

### 回归与评审（EV）
- [ ] `P3-EV-01` 优化后全量回归评测：确认 Accuracy +≥25pp、Recall ≥85% 最终达标（AC-1/2）
- [ ] `P3-EV-02` 分诊开启前后对比：确认 Fast Path 不损伤整体效果
- [ ] `P3-EV-03` G3 评审材料：效果报告 + 压测报告 + 护栏测试报告 + 增量更新演练记录

## 交付物

1. 达标的生产候选版本（效果/延迟/成本三达标）
2. 压测报告、护栏专项测试报告、增量更新演练记录
3. 监控看板 + 预算熔断机制
4. G3 评审结论
