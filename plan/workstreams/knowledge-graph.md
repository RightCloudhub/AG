# 工作流：知识图谱构建与治理

**覆盖需求**：FR-KG-01 ~ 07 · **相关阶段任务**：P1-KG-*、P2-KG-*、P3-KG-*
**负责人**：知识图谱方向（1-2 人）

## 1. 管线总览

```
文档接入 → 清洗分段 → LLM抽取(三元组+置信度+引用) → Schema校验
  → 置信度过滤 → 冲突检测 → 实体消歧 → 合并入图 → 索引同步(向量/全文)
                     ↓ 低置信/冲突
                人工审核队列 → 审核通过入图
```

## 2. 关键设计

### 2.1 Schema 管理（FR-KG-03）
- Schema 以版本化配置文件定义：实体类型（含属性约束）、关系类型（含头尾实体类型约束）。
- 抽取输出必须通过 Schema 校验才能入图；不合规项落"拒绝表"供分析（常见拒绝原因反哺 Schema 迭代）。
- Schema 变更走评审：新增类型向后兼容；删除/改名类型需评估存量数据迁移。

### 2.2 抽取（FR-KG-02）
- 主路径：LLM 结构化输出（JSON Schema 约束），每条三元组带 `confidence`、`source_doc_id`、`source_span`。
- 兜底：规则/词典抽取覆盖高频确定性模式（如公司后缀、职务词表），与 LLM 结果合并去重。
- 质量基线：POC 期抽检正确率 ≥70%，MVP 期通过 Prompt 迭代 + 置信度阈值过滤提升至 ≥85%（入图后口径）。

### 2.3 实体消歧（FR-KG-04，阶段三）
三级策略，逐级升级：
1. **规则归一**：大小写/全半角/常见别名词典。
2. **候选生成**：实体名 embedding 相似度 + 图上下文（邻居重叠度）召回候选对。
3. **LLM 判定**：候选对 + 各自上下文 → 合并/不合并/不确定；"不确定"进入人工审核队列。

合并操作可追溯、可撤销（保留合并前实体快照）。

### 2.4 增量更新（FR-KG-05，阶段三）
- 新文档以批次为单位处理；冲突检测比对存量图：同头尾实体的同类关系值不同 → 冲突。
- 冲突处理策略：新来源置信度显著更高 → 自动更新并保留旧值历史；否则入审核队列。
- 索引同步：入图事务完成后触发向量/全文索引增量更新与相关缓存失效。
- 线上不中断：写入走批次事务，查询读快照；演练验证见 P3-KG-01 / AC-5。

### 2.5 质量治理闭环（FR-KG-06）
- 每批次抽取后随机抽样 N 条入审核界面，人工标注对/错。
- 抽检正确率、按关系类型的错误分布进入周报；错误高发类型触发 Prompt/规则迭代。
- 审核结论积累为抽取评测集，Prompt 变更须过此集回归。

## 3. 数据模型要点

| 对象 | 关键字段 |
|---|---|
| 实体 | id、类型、规范名、别名列表、属性、来源引用列表、创建/更新时间 |
| 关系 | id、类型、头/尾实体、属性、confidence、来源引用列表、状态（active/pending_review/rejected） |
| 来源引用 | doc_id、span 定位、抽取批次 id |
| 审核项 | 类型（抽取抽检/消歧/冲突）、载荷、状态、审核人、结论 |

## 4. 风险与对策（本工作流特有）

| 风险 | 对策 |
|---|---|
| 抽取噪声拖垮下游效果 | 置信度阈值过滤 + 抽检闭环；效果归因时单列"图谱缺失/错误"类 badcase |
| 消歧错误合并（不同实体被合并） | 合并可撤销；高影响实体（高度数节点）合并强制人工确认 |
| Schema 过早固化 | POC 期 Schema 标记 V0 可破坏性变更；MVP 后才承诺兼容 |
