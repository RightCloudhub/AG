# 工作流：混合检索

**覆盖需求**：FR-RT-01 ~ 05 · **相关阶段任务**：P1-RT-*、P2-RT-*、P3-PERF-02/03/04
**负责人**：检索系统工程（1 人）

## 1. 三路检索 + 融合

```
子问题 ──┬─ 向量检索(chunk embedding, Top-K)
         ├─ 图检索(k跳邻居/路径/子图, 剪枝+Top-K路径)   ─→ RRF融合 ─→ 统一候选列表
         └─ 全文检索(BM25)                                      (预留Re-ranker接口)
```

三路并行执行（阶段三 P3-PERF-02），任一路失败降级为其余两路结果并记录告警，不整体失败。

## 2. 统一候选项契约（P2-RT-02）

所有检索工具返回统一结构，供融合排序与 Agent 消费：

| 字段 | 说明 |
|---|---|
| type | vector_chunk / graph_path / graph_neighbor / fulltext_chunk |
| content | 文本内容或路径的自然语言化描述 |
| structured | 图类结果的结构化载荷（节点/边序列） |
| score | 该路内部分数（不跨路比较，融合用排名） |
| citations | 来源引用（doc_id + span，或边的来源引用） |

## 3. 各路要点

### 3.1 向量检索（FR-RT-01）
- chunk 策略：按语义段落切分，目标 200-500 token，重叠可配置；chunk 记录 doc_id + span。
- Top-K 可配置（默认 10）；embedding 结果缓存（P3-PERF-04）。

### 3.2 图检索（FR-RT-02）—— 本工作流核心
封装为三个工具供 Executor 调用：
1. **邻居查询**：实体 → k 跳邻居子图（默认 k=1，按关系类型过滤）
2. **路径查询**：两实体间 Top-K 最短/加权路径（k≤4 跳）
3. **子图遍历**：种子实体集 + 关系类型约束 → 相关子图

**路径爆炸防护**（PRD 难点表对应）：
- 邻居扩展按"关系与子问题的相关性"剪枝：关系类型 embedding 与子问题相似度低于阈值则不扩展
- 每层扩展节点数上限（默认 50）、路径总数 Top-K 采样（默认 20）
- 高度数节点（degree > 阈值）扩展降级：只取与子问题相关的关系类型

**注入防护**（NFR-07）：Cypher/GQL 一律参数化模板拼装，实体名/类型作为参数传入，禁止字符串拼接查询。

### 3.3 全文检索（FR-RT-03）
- BM25，覆盖专有名词、代号、精确短语场景；与向量检索共用 chunk 存储。

## 4. 融合排序（FR-RT-04，阶段三）

- **V1：RRF**（Reciprocal Rank Fusion）：`score = Σ 1/(k + rank_i)`，k=60 起步，按评测调参。规避三路分数量纲不一致问题。
- **V2 预留：学习型 Re-ranker**：接口设计为 `rerank(query, candidates) → sorted candidates`，训练数据来自评测集金标证据 + 线上反馈（阶段五 P5-CAP-04 落地）。
- 融合层保序输出引用，供答案生成绑定。

## 5. 缓存策略（P3-PERF-04）

| 缓存 | Key | 失效 |
|---|---|---|
| embedding | 文本 hash | 不失效（内容寻址） |
| 子问题检索结果 | 子问题规范化 hash + 索引版本 | 图谱/索引更新递增版本号自然失效 |
| 热点问题答案 | 原问题规范化 hash + 索引版本 | 同上 + TTL |

## 6. 评测口径

- 检索层独立评测：金标证据在融合后 Top-N 内的命中率（Recall@N），与端到端 AC-2 区分，便于归因。
- badcase 归因中"检索失败"细分：该路未召回 / 融合排序沉底 / 图谱缺失（转 KG 工作流）。
