# 测试策略

**目标**：覆盖率 ≥80%（NFR-09），TDD 工作流（红→绿→重构）自 MVP 阶段起强制执行；POC 代码豁免但核心契约（推理链 JSON、候选项契约、存储接口）必须带测试。

## 1. 测试分层

| 层 | 范围 | 工具/方式 | 要点 |
|---|---|---|---|
| 单元测试 | 各模块纯逻辑 | pytest + mock | LLM/存储全部 mock；重点：护栏触顶逻辑、Memory 去重、RRF 计算、Schema 校验、引用绑定拦截 |
| 集成测试 | 模块间 + 真实存储 | pytest + testcontainers（Neo4j/向量库/ES 容器） | 图检索剪枝正确性、增量更新事务、审核队列流转、API envelope 与错误码 |
| E2E 测试 | API 端到端 | 固定小图谱 fixture + LLM 录制回放（record/replay） | 关键用户流：多跳问答、无答案兜底、预算熔断、SSE 流式、审计回查 |
| 评测回归 | 效果指标 | evals/run.py | 与功能测试分离；见 [evaluation.md](../workstreams/evaluation.md) |

## 2. LLM 相关测试的特殊约定

- **确定性优先**：单元/集成/E2E 一律不打真实 LLM——用录制回放或规则 stub，保证 CI 稳定与零成本。
- **结构化输出健壮性**：对每个 LLM 交互点测试"格式错误响应"路径（解析失败→重试→降级）。
- **Prompt 变更 ≠ 代码变更**：Prompt 修改不跑单测门禁，但必须跑评测集回归（dev 集）后合入。
- **图节点可测性（ADR-005）**：Planner/Executor/Critic 节点实现为"状态入→状态出"的纯函数，单测不依赖 LangGraph 运行时；编译后的完整 StateGraph 用 stub LLM 跑集成测试（条件边路由、checkpoint 恢复、事件流顺序）。

## 3. 关键测试场景清单（须覆盖）

### Agent 循环
- [ ] 触达最大跳数/调用次数/预算 → 正确兜底、状态 `partial`/`no_answer`
- [ ] recursion_limit 触达（模拟自研护栏失效）→ 仍正确兜底并产生告警事件
- [ ] Critic 返回重复子问题 → Memory 拒绝、强制换方向
- [ ] 子问题 DAG 依赖：占位子问题在依赖满足后具体化
- [ ] Fast Path 证据不足 → 升级 Agentic（且仅一次）

### 检索
- [ ] 高度数节点扩展降级；每层扩展上限生效
- [ ] 一路检索故障 → 两路降级 + 告警，不整体失败
- [ ] Cypher 参数化：实体名含引号/特殊字符不产生注入（NFR-07 安全用例）

### 知识管线
- [ ] 不合规三元组拒绝入图并落拒绝表
- [ ] 增量更新冲突：高置信自动更新保留历史 / 低置信入审核
- [ ] 消歧合并可撤销，快照恢复

### API
- [ ] 入参校验 fail fast（超长问题、非法参数）
- [ ] 预算/速率超限的错误码与不泄露内部细节的 message
- [ ] 租户隔离：A 租户查不到 B 租户文档与图数据

### 诚实性（AC-7 支撑）
- [ ] 无引用论断被生成层拦截
- [ ] 无答案 fixture 问题 → 固定兜底结构输出

## 4. 覆盖率与门禁

- CI 门禁：单测+集成覆盖率 ≥80%，低于阈值阻塞合入（见 [cicd-observability.md](./cicd-observability.md)）。
- 覆盖率统计排除：`configs/`、`scripts/`、生成代码。
- 每个 bugfix 附回归测试（先写失败测试复现，再修）。

## 5. 安全检查

上线前（P4-REL-02）security-reviewer 全量检查，日常遵循安全规范清单：
- 无硬编码密钥；启动时校验必需环境变量
- 全部外部输入 schema 校验；Cypher/SQL 参数化
- Prompt 注入基础防护：用户输入与系统指令隔离、检索内容标记为不可信数据
- 鉴权/速率限制/租户隔离测试用例常绿
- 错误信息不含堆栈/内部路径
